# 功能規格：Popup 截圖功能

**功能分支**：`004-popup-screenshot`  
**建立日期**：2026-04-26  
**狀態**：草稿  
**輸入**：使用者描述："做popup頁面按下相機icon後就會觸發網頁截圖功能，可以自行選擇要截圖的大小自行拉動，截圖後暫存到 popup裡面，如果關閉Popup就刪除，或是自己按右上角圖片刪除"

## 使用者情境與測試 *(必填)*

### 使用者故事 1 - 截圖拍攝 (優先順序：P1)

使用者在 popup 頁面點擊相機圖示，系統會對當前分頁進行網頁截圖，並允許使用者透過拖曳選取框來選擇要截取的區域。

**為何此優先順序**：這是核心功能。若無法截圖，整個功能就無法運作。

**獨立測試**：可通過點擊相機圖示，驗證截圖選取介面是否正確顯示，並完成選取操作。

**接受情境**：

1. **假設** popup 頁面已開啟，**當**使用者點擊相機圖示，**則**系統開始截圖並顯示選取介面
2. **假設** 選取介面已顯示，**當** 使用者在選取框內拖曳，**則** 選取框范圍隨之調整
3. **假設** 選取框正在調整，**當** 使用者在邊緣/角落拖曳調整大小，**則** 選取框尺寸隨之改變而位置保持
4. **假設** 使用者已完成選取並確認，**當** 使用者點擊確認按鈕或按下 Enter，**則** 截圖保存至 popup 顯示
5. **假設** 選取介面已顯示，**當** 使用者按下 Escape 鍵，**則** 選取取消並關閉選取介面

---

### 使用者故事 2 - 截圖預覽與刪除 (優先順序：P2)

截圖完成後，使用者可以在 popup 中預覽截圖，並可隨時手動刪除。

**為何此優先順序**：使用者需要能夠檢視和管理已拍攝的截圖。

**獨立測試**：可通過完成截圖後，驗證截圖是否正確顯示，並測試刪除功能。

**接受情境**：

1. **假設** 截圖已完成並顯示在 popup，**當** 使用者將鼠標移至截圖右上角，**則** 刪除按鈕可見
2. **假設** 刪除按鈕可見，**當** 使用者點擊刪除按鈕，**則** 截圖立即從 popup 移除
3. **假設** ClothingAreaCard 已顯示截圖，**當** 使用者點擊截圖，**則** 截圖可放大或進入編輯模式

---

### 使用者故事 3 - 自動刪除機制 (優先順序：P3)

關閉 popup 時，截圖自動被刪除，確保持久化存儲的清潔。

**為何此優先順序**：防止截圖資料殘留，保護使用者隱私。

**獨立測試**：可通過關閉 popup 後重新開啟，驗證截圖是否已被清除。

**接受情境**：

1. **假設** popup 中已有截圖，**當** 使用者關閉 popup（點擊外部或點擊關閉），**則** 截圖從記憶體和臨時存儲中刪除
2. **假設** 使用者重新開啟 popup，**則** 先前拍攝的截圖不再存在

---

### 邊界情況

- 當 `chrome.tabs.captureVisibleTab` API 呼叫失敗時，系統顯示錯誤提示並保持選取介面，讓使用者可重試
- 選取框拖曳或調整大小時，必須完全保持在視窗範圍內
- 當選取區域小於最小尺寸時，系統顯示警告提示，但允許使用者確認並截圖
- 當使用者在選取過程中按下 Escape 鍵，取消選取並關閉選取介面，回到原始 popup 狀態

## 需求 *(必填)*

### 功能需求

- **FR-001**：系統必須在使用者點擊相機圖示時，透過 `chrome.tabs.captureVisibleTab` API 截取當前分頁
- **FR-002**：系統必須在截圖上覆蓋半透明遮罩，並顯示可調整大小的選取框
- **FR-003**：使用者必須能夠透過拖曳選取框內部來移動選取區域
- **FR-004**：使用者必須能夠透過拖曳選取框邊緣或角落來調整選取區域大小
- **FR-005**：系統必須在使用者確認後，將選取區域的圖片資料以 Blob 格式暫存於記憶體中
- **FR-006**：系統必須在 popup 關閉時，自動清除暫存的截圖 Blob 資料
- **FR-007**：系統必須在使用者點擊截圖右上角的刪除按鈕時，立即移除該截圖
- **FR-008**：選取框必須具有最小尺寸限制（例如 50x50 像素），以防止過小的選取
- **FR-009**：系統必須在選取過程中顯示選取區域的即時尺寸提示（寬度 x 高度）
- **FR-010**：使用者按下 Escape 鍵時，系統必須取消選取並關閉選取介面
- **FR-011**：選取框拖曳或調整大小時，必須完全保持在視窗範圍內
- **FR-012**：當選取區域小於最小尺寸時，系統顯示警告提示，但允許使用者確認

### 關鍵實體 *(若功能涉及資料)*

- **截圖資料 (ScreenshotData)**：包含 Blob 資料、時間戳記，用於記憶體暫存，生命周期與 popup 一致
- **選取框狀態 (SelectionState)**：包含位置座標 (x, y) 和尺寸 (width, height)，用於追蹤選取區域
- **遮罩層 (OverlayMask)**：覆蓋在截圖上的半透明層，用於突顯選取區域

## 成功標準 *(必填)*

### 可衡量成果

- **SC-001**：使用者在點擊相機圖示後，截圖選取介面在 1 秒內顯示
- **SC-002**：選取框拖曳操作流暢，延遲不超過 16ms（60fps）
- **SC-003**：使用者可在 3 次點擊內完成截圖（點擊相機 → 拖曳選取 → 確認）
- **SC-004**：popup 關閉後，重新開啟時截圖不存在（自動清除率 100%）
- **SC-005**：刪除按鈕可在滑鼠懸停後 100ms 內響應

## 假設

- 使用者已授予擴充功能「網站內容」和「分頁」相關權限
- 截圖來源為當前作用中的分頁（active tab）
- 選取框預設尺寸為截圖的 50%，預設位置為截圖中央
- 最小選取尺寸為 50x50 像素，以確保有用的截圖區域
- 刪除按鈕使用 X 圖示，放置於截圖右上角
- 不需要將截圖持久化到 IndexedDB（僅記憶體暫存）
